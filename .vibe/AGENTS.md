# Repository Agent Instructions

These instructions apply specifically to the gorzko.github.io repository.

## Content Sourcing

- **Wikimedia Commons Only**: All image content must be sourced from Wikimedia Commons with proper attribution. Do not use images from other sources without explicit approval.

- **Attribution**: Maintain proper attribution metadata for all Wikimedia-sourced content. Include author, license, and source URL information.

## External Resources

- **Validation Required**: All external resources must be downloaded with proper validation and rate limiting. Never assume downloaded content is valid without verification.

- **Binary Verification**: All binary files (JPG, PNG) must have non-zero size before commit. Verify files are not corrupted and are stored in the correct binary format.

## Image Requirements

For detailed image quality standards, see `img/image-quality.md`.

### Summary of Key Requirements
- **Photographs only**: No illustrations, drawings, or digital art
- **Minimum Resolution**: 1200x900 pixels for standard images (desktop support)
- **Minimum File Size**: 100KB standard, 500KB+ preferred
- **Representative**: Adult specimens in standard pose showing typical appearance
- **Quality**: Sharp, in focus, good lighting, clear composition
- **Formats**: JPG (preferred), PNG (transparency only), SVG (vector only)

### Responsive Images
- Provide at least 2 sizes for responsive design (800px mobile, 1920px desktop)
- Consider retina/HiDPI versions at 2x resolution

### Selection Criteria
- Adult specimens preferred (not juveniles/hatchlings)
- Standard, natural pose (not unusual behaviors)
- Clear view of identifying features
- Natural, accurate colors

For current images requiring attention, see `img/image-quality.md` - Known Issues & Corrections section.
